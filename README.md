## Сборка
Проект собран на **Ubuntu 20.04.6 LTS**. Для запуска из корня проекта:
```shell
./build/tape_sort <input_tape> <output_tape>
./build/tape_sort --generate <input_tape> <size>

./build/tape_test
```
Вы можете самостоятельно собрать проект с помощью `CMake`. Для сборки требуется:
- Компилятор с поддержкой С++17 (GCC, Clang, MSVC)
- CMake 3.10+
- Boost.Test (для unit-тестирования)
#### Linux
```bash
sudo apt install libboost-test-dev

mkdir build && cd build
cmake ..
make
```
#### Windows (MSVC)
```shell
mkdir build
cd build
cmake ..
cmake --build . --config Release
```
## О приложении

Программа запускается следующим образом
```shell
./build/tape_sort <input_tape> <output_tape>
./build/tape_sort <input_tape> <output_tape> <memory_size>
./build/tape_sort --generate <input_tape> <size>

./build/tape_test
```
- С флагом `--generate` приложение генерирует файл с случайными числами и его можно использовать для сортировки
- Сортировка принимает входной и выходной файлы (и ограничение оперативной памяти). Если выходного файла не существует, он будет создан. Также она выводит время, затраченное на сортировку и дает возможность вывести в строку все отсортированные числа. С задержками на это может уйти время. Измерения времени сортировки (с задержками по умолчанию и без)

| Количество элементов в ленте | Ограничение памяти | Задержки операций | Время (в секундах) |
| ---------------------------- | ------------------ | ----------------- | ------------------ |
| 200                          | нет                | нет               | 0.3                |
| 200                          | 100                | нет               | 0.4                |
| 400                          | нет                | нет               | 0.13               |
| 400                          | 100                | нет               | 0.11               |

- Тесты проверяют корректность работы файловой ленты и сортировки (без искусственного замедления)
## Файловая лента

В приложении реализован класс магнитной ленты `FileTape`, которая использует файлы в качестве хранилища данных. Реализует интерфейс `Tape`

```cpp
FileTape(const std::string & filepath, const std::string & config);
```
Конструктор принимает `filepath` - файл магнитной ленты, `config` - `JSON` конфиг для создания искусственных задержек
Дальше необходимый формат `JSON`. Если он не будет соответствовать формату, программа применит задержки по умолчанию. Эти задержки представлены на примере `JSON`:
```json
{
  "delays": {
    "read": 5,
    "write": 5,
    "rewind": 30,
    "move": 2
}
```
Все задержки **измеряются в миллисекундах**

```cpp
  void read(int & value) override;
```
Читает ячейку ленты и записывает в `value`. Чтение при активном `end()` - неопределенное поведение.

```cpp
  void write(int value) override;
```
Записывает `value` в ячейку и расширяет файл ленты
  
```cpp
  void next() override;
```
Перемещается на ячейку вперед 

```cpp
  void prev() override;
```
Перемещается на ячейку назад

```cpp
  void rewind() override;
```
Возвращает "головку считывателя" в начало

```cpp
  bool end() override;
```
Проверяет закончилась ли лента

Существует фабрика `TempFileTapeCreator`, способная создавать временные ленты и необходимая для работы сортировки. Реализует интерфейс `TempTapeCreator`

Конструкторы:
```cpp
TempFileTapeCreator() = default;
TempFileTapeCreator(const std::string & configuration);
```
Использует `configuration` для `config` у `FileTape`. Все временные ленты, созданные на этой фабрике будут использовать этот конфигурационный файл и иметь соответствующие задержки. Конструктор по умолчанию пробует использовать конфигурационный файл по пути `configs/default_delays.json`.

```cpp
~TempFileTapeCreator()
```
Все созданные на этой фабрике временные ленты удаляются при разрушении экземпляра фабрики

```cpp
std::unique_ptr< Tape > create() override;
```
Создает экземпляр временной ленты. Он будет хранится в папке `tmp` (не ОС, а в проекте) и иметь уникальное имя 
## Сортировка

```cpp
TapeSorter(size_t memory_size = 100);
```
Создает класс для сортировки. Можно указать ограничение по использованию оперативной памяти. Внутри сортировка сможет читать одновременно до `memory_size` элементов ленты

```cpp
template< typename Compare = std::less< int > >
void sort(Tape * src, Tape * dst, TempTapeCreator * creator = nullptr, Compare comp = Compare());
```
- Принимает `src` ленту `Tape` и записывает отсортированные данные на ленту `dst`. На выходе `rewind()` к `src` не применен, а к `dst` уже применен
- Если объем оперативной памяти большой и данные на ленте умещаются в нее, они будут отсортированы через `std::sort` и быстро возвращены. Если заранее известно, что объем ленты меньше объема оперативной памяти, можно не передавать фабрику `TempTapeCreator`
- Если данных больше, чем вмещает оперативная память, то будут созданы временные ленты и к ним будет применена многофазная сортировка слиянием *O(NlogN)*
- Принимает компаратор для сравнения элементов. По умолчанию `std::less< int >`
